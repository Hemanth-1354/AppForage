⚡ AppForge Pipeline Architecture

Natural Language → Intent → Architecture → Schema → Refinement → Validation → Execution

Stage 1
Intent Extraction
Parses NL into structured intermediate form. Extracts entities, features, roles. Flags ambiguities and makes documented assumptions.
→ entities, features[], roles[], auth_required, assumptions[]
Stage 2
Architecture Design
Converts intent into full application architecture. Defines pages, API resource groups, data entities with relations, auth model.
→ pages[], api_groups[], data_entities[], auth_model
Stage 3
Schema Generation
Generates 4 complete schemas: UI (pages, components, data bindings), API (endpoints, methods, auth), DB (tables, columns, FKs), Auth (roles, permissions, middleware).
→ ui_schema + api_schema + db_schema + auth_schema
Stage 4
Refinement Layer
Audits cross-layer consistency: UI data_sources → API endpoints, API fields → DB columns, roles everywhere → auth schema. Applies targeted fixes.
→ corrected schema + issues_found[] + fixes_applied[]
Stage 5
Validation Engine
Produces quality score + blocking errors + warnings + repairable list. Triggers Auto-Repair for targeted fixes without full retry.
→ valid:bool, score:0-100, errors[], warnings[], repairables[]
Runtime
Execution Simulator
Proves output is executable: simulates DB table creation, auth token issuance, API routing, UI page rendering. Generates schema.sql + routes.js + routes.jsx.
→ executable:bool + code artifacts
